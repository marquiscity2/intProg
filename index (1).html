1. Name two differences between netlify and back4APP, and provide 3 alternatives and competitors to them.

Netlify primarily focuses on web hosting and continuous deployment, offering features like serverless functions, form handling, and asset optimization. In contrast, back4app provides a complete backend-as-a-service solution, including a database, user authentication, cloud functions, and APIs.

Netlify is aimed at web developers who want to deploy static websites and web applications quickly and efficiently. Back4app is designed for mobile app developers who want to focus on the front-end development of their app and delegate backend tasks to a third-party service.

A compound selector is a selector that combines two or more selectors, allowing you to target elements that match both selectors.

An attribute selector, on the other hand, is a selector that targets elements based on the presence or value of an attribute.

3. Briefly compare json, xml, yaml and markdown.

JSON is a lightweight format based on JavaScript and widely used for APIs, configuration files, and data storage. It uses a key-value pair format, making it easy to parse and generate using programming languages.

XML is a flexible markup language used for storing and transporting data in web applications. Users can define their own tags and structure, and it can be parsed and generated using various programming languages.

YAML is a human-readable data serialization format used for configuration files, data exchange, and storing data. It has a structure similar to JSON but is more readable and easier to write.

Markdown is a lightweight markup language used to format text with simple syntax. It is often used for creating easy-to-read text documents that can be converted into HTML or other formats, and is commonly used in documentation and README files.

5. Differentiate javascript, typescript, transpile , ES6/7, node. Is Node a framework? Justify your answer.

JavaScript is a programming language used for web development, both on the client-side (in web browsers) and server-side (using technologies like Node.js). It is a dynamically-typed language, meaning that variable types are determined at runtime.

TypeScript: TypeScript is a superset of JavaScript, which means it extends the language with additional features and syntax. It provides static typing and compile-time type checking, which helps catch errors earlier in the development process. TypeScript is transpiled (compiled) to JavaScript before being run in a web browser or on a server.

Transpile: Transpiling is the process of converting one programming language into another. For example, TypeScript is transpiled into JavaScript. Transpiling is typically done to make code written in one language compatible with another environment or to add additional features to a language.

ES6/7: ES6 (ECMAScript 6) and ES7 (ECMAScript 7) are newer versions of the JavaScript language. They introduced new features and syntax, such as arrow functions, classes, and the spread operator. These features are now widely used in modern JavaScript development.

Node: Node.js is a runtime environment for JavaScript, which allows developers to run JavaScript code on a server, outside of a web browser. It provides a set of built-in modules and libraries for server-side development. Node.js also supports asynchronous programming, which allows for more efficient handling of I/O operations.

No, Node.js is not a framework. It is an open-source, cross-platform, server-side runtime environment for executing JavaScript code.

9. Explain npm, npx, nvm, package.json and package-lock.json files

npm (short for Node Package Manager) is a package manager for the Node.js runtime environment. It is used to install, manage, and share packages (libraries, frameworks, tools, etc.) written in JavaScript.

npx is a package runner tool that is included with npm. It is used to execute command-line tools and packages that are installed locally or published on the npm registry, without the need to install them globally.

nvm (short for Node Version Manager) is a tool that allows developers to install and manage multiple versions of the Node.js runtime environment on the same machine. This is useful for testing and compatibility purposes, as different versions of Node.js may have different features and behaviors.

package.json is a configuration file that is used to describe a Node.js project and its dependencies. It includes metadata such as the project name, version, author, license, and scripts, as well as a list of dependencies and their versions.

package-lock.json is a file that is automatically generated by npm when a project is installed or updated. It includes a complete list of all the dependencies and their exact versions that are installed in the project, as well as their transitive dependencies. This ensures that the same versions of the dependencies are installed on all machines, and avoids version conflicts and unexpected changes in behavior.

10. Differences among CSS, LESS, SASS (Syntactically Awesome Stylesheet)

SASS (Syntactically Awesome Stylesheet) is a CSS pre-processor, which helps to reduce repetition with CSS and saves time. It is more stable and powerful CSS extension language that describes the style of document structurally. This tutorial covers the basics of SASS.

CSS: CSS (Cascading Style Sheets) is the standard stylesheet language used on the web. It uses a simple syntax to define styles for HTML elements, including properties such as color, font, padding, and margin. LESS: LESS is a dynamic stylesheet language that extends the functionality of CSS. It uses variables, functions, mixins, and other features to make CSS more reusable and maintainable. LESS code is processed on the server side into CSS code, which is then sent to the client's browser.

12. Define the following concepts: nosql, Mongodb, collection, document, rdbms, mysql, sql, MongoDB Query Language (MQL)

NoSQL: A type of database that does not use the traditional relational database model and typically stores data in a non-tabular format, making it more flexible and scalable for handling large volumes of unstructured or semi-structured data.

MongoDB: A popular open-source NoSQL document-oriented database that stores data in JSON-like documents with dynamic schemas. It supports flexible querying and indexing, making it a popular choice for web applications and other use cases that require high scalability and performance.

Collection: In MongoDB, a collection is a group of related documents stored in a specific database. It is analogous to a table in a relational database and can contain any number of documents.

Document: In MongoDB, a document is a set of key-value pairs stored in a collection. It is analogous to a row in a table in a relational database, but with a more flexible schema. Documents can have different fields and data types, making them well-suited for storing unstructured or semi-structured data.

RDBMS: Relational Database Management System is a type of database management system that uses a relational model for data organization and manipulation.

MySQL: A popular open-source RDBMS that uses SQL as its query language. It is known for its reliability, scalability, and ease of use, making it a popular choice for web applications and other use cases.

SQL: Structured Query Language is a programming language used to manage relational databases. It is used to create, read, update, and delete records in a database.

MongoDB Query Language (MQL): A query language used in MongoDB to retrieve and manipulate data in MongoDB collections. It is similar to SQL in syntax but uses JSON-like structures to query the document-oriented data in MongoDB.

Data attributes allow us to store extra information on standard, semantic HTML elements without other hacks such as non-standard attributes, or extra properties on DOM. Any attribute on any element whose attribute name starts with data- is a data attribute.

Media Queries is a CSS technique that allows web developers to apply different styles to a web page based on the characteristics of the device that is used to access it. Viewport is the visible area of a web page in a web browser and Grid Layout is a CSS technique that allows web developers to create complex layouts with grids of columns and rows.

15. Asynchronous, await, vs. synchronous, blocking calls vs. noblocking calls

Synchronous code runs sequentially, blocking the program until each line of code has finished executing. Asynchronous code, however, allows the program to continue running while a particular task is being executed, making it useful for tasks that may take a long time to complete.

A blocking call is a synchronous operation that stops the program from executing any other code until the operation is complete. During this time, the program is essentially "blocked" from doing anything else and a non-blocking call, on the other hand, is an asynchronous operation that allows the program to continue executing other tasks while waiting for the operation to complete in the background.

16. What is restful services? What is restful API endpoints? What is routing? restful, restful api, restful service, client, clean

RESTful services are web services that follow the principles of Representational State Transfer (REST) architectural style.

RESTful API endpoints are URLs or routes that are used to interact with a web application through RESTful web services. They allow clients to perform CRUD (Create, Read, Update, and Delete) operations on resources.

Clean URLs (also known as user-friendly URLs, pretty URLs, search engine-friendly URLs or RESTful URLs) are web addresses or Uniform Resource Locator (URLs) intended to improve the usability and accessibility of a website, web application, or web service by being immediately and intuitively meaningful to non-expert users.

A resource is the conceptual entity being represented by the API, while an endpoint is the specific URL used to access or manipulate that resource.

17. Fetch, ajax vs. promise, callback function, ascychronous function

Fetch and AJAX are methods to make asynchronous HTTP requests. Promises represent a value that may not be available yet, allowing to attach callbacks to execute when the value is available. Callback functions handle asynchronous operations by passing them as arguments to other functions. Asynchronous functions return a Promise and use the await keyword to pause execution until the Promise is resolved, making the code look like synchronous code.

18. Express, Express routing, express middleware

Express is a popular web application framework for Node.js, used for building server-side applications, APIs, and backend services.

Express routing is the process of defining the application's endpoints using app.get(), app.post(), app.put(), app.delete(), and other methods, and each route has a handler function.

Express middleware is a set of functions used to modify incoming requests or outgoing responses, and can be used to perform tasks such as authentication, logging, error handling, and more. Middleware functions are defined using app.use() or app.METHOD() methods and are executed in a specific order for each incoming request.

19. What are the following? Briefly compare them storybook,cypress,. jest, mocha, jasmine

Storybook is a tool used for building UI components in isolation, allowing developers to develop and test UI components in isolation before integrating them into a larger application. Cypress is an end-to-end testing framework that is used for testing web applications.

Cypress allows developers to write tests that simulate user actions, such as clicking buttons or filling out forms, and then verifies that the application responds as expected.

Jest is a JavaScript testing framework that is used for unit testing, integration testing, and snapshot testing. Jest is widely used and has a large community, making it easy to find support and resources.

Mocha is another JavaScript testing framework that is used for unit testing, integration testing, and functional testing. Mocha is highly customizable and flexible, allowing developers to use their preferred assertion libraries and other testing tools.

Jasmine is a behavior-driven testing framework for JavaScript that is used for unit testing, integration testing, and functional testing. Jasmine is designed to be easy to read and write, making it a good choice for teams with varying levels of experience with testing.

Basically, storybook is focused on building UI components in isolation, while Cypress, Jest, Mocha, and Jasmine are all testing frameworks that are used for testing web applications, but with different focuses and features. Cypress is an end-to-end testing framework, Jest and Mocha are widely used JavaScript testing frameworks, and Jasmine is a behavior-driven testing framework.

22. What is same-origin policy?

The same-origin policy (SOP) is an important concept in the web application security model. Under the policy, a web browser permits scripts contained in a first web page to access data in a second web page, but only if both web pages have the same origin.

23. react Hooks allow function components to have access to state and other React features (such as lifecycle methods). Briefly explain the purpose of each of the following hooks: useState, useEffect, useContext, useRefuse, Reducer, useCallback useMemo.

The useState hook is used to manage state within a functional component

The useEffect hook is used to manage side effects within a functional component.

The useContext hook is used to access context data from a parent component within a child component.

The useRef hook is used to create a mutable reference that persists between renders of a component.

The useReducer hook is used to manage state within a functional component in a more complex manner.

The useCallback hook is used to memorize a function so that it only gets re-created when certain dependencies change.

The useMemo hook is used to memorize a value so that it only gets re-computed when certain dependencies change.

24. Session and sessionStorage, localStorage, IndexedDB

Session and sessionStorage are both used to store data temporarily on the client-side. Session storage stores data for the current session only, while sessionStorage stores data for a single page session.

localStorage is also used to store data on the client-side but unlike session and sessionStorage, the data is persistent and remains even after the user closes the browser window.

IndexedDB is a more powerful client-side storage option that provides a more structured way to store data, similar to a database.

25. Provide answers for the following concepts: client-server application, socket programming, ip address, tcp port, http

A client-server application is a software architecture where a client sends requests to a server over a network, and the server responds with data or performs actions on behalf of the client.

Socket programming is a way of connecting two nodes on a network to communicate with each other.

An IP address is a unique numerical identifier assigned to each device connected to a computer network.

A TCP port is a number used to identify a specific network application or service on a computer.

HTTP is a protocol used for transferring data over the World Wide Web.
